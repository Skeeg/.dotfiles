# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Purpose

Personal ZSH/Bash dotfiles managed via symlinks. Running `bootstrap.sh` creates symlinks from `$HOME` to files in this repo, with automatic backup of any conflicting files. The repo is designed to live at `~/.dotfiles` but can be placed anywhere.

## Key Commands

```bash
# Bootstrap: symlink dotfiles into $HOME
./bootstrap.sh

# Merge git aliases into ~/.gitconfig without overwriting existing config
./merge_gitconfig.sh

# Install work dependencies via Homebrew
brew bundle --file=.personal/WorkBrewfile

# Install personal dependencies via Homebrew
brew bundle --file=.personal/PersonalBrewfile

# Lint shell scripts (ShellCheck is in the Brewfile)
shellcheck bootstrap.sh merge_gitconfig.sh
shellcheck .profile.d/*.sh .profile.d/macos/*.sh
```

## Architecture

### Symlink Strategy

`bootstrap.sh` symlinks everything in the repo root to `$HOME` except: `.git/`, `bootstrap.sh`, `merge_gitconfig.sh`, `readme.md`, `CLAUDE.md`, and the `.personal/` folder contents. The `.profile.d/merge_content/` subdirectory is also excluded — it holds git alias definitions used only by `merge_gitconfig.sh`.

### Directory Clobbering Risk

Bootstrap symlinks **entire root-level directories** as a single symlink, not file-by-file. This has two important consequences:

**Safe — adding files inside an already-tracked directory:**
If a directory like `.config/` already exists in the repo and is symlinked as `~/.config`, adding a new subdirectory (e.g., `.config/claude/`) requires no re-bootstrap — the new path is immediately accessible via the existing symlink. You do need to explicitly allow the new subdirectory in `.config/.gitignore`, which uses an allowlist pattern (`*` + `!exceptions`).

**Dangerous — adding a root-level item that shadows an existing `$HOME` directory:**
If you add `.claude/` at the repo root, bootstrap will replace `~/.claude` (which contains Claude Code's state, history, and settings) with a symlink to the repo directory — **destroying all existing content** after backing it up. The same risk applies to any directory that already exists in `$HOME` (e.g., `.ssh/`, `.gnupg/`).

**Rule of thumb:** place config files for tools that manage their own state directories (Claude Code, SSH, GPG, etc.) inside `.config/<tool>/` rather than at the repo root. Use plugin scripts (`.profile.d/`) or helper functions (e.g., `start-claude`) to copy or apply those files at runtime rather than symlinking them directly.

### Plugin System (`.profile.d/`)

Shell functionality is split into per-tool plugin files (`*.plugin.sh`). Both `.zshrc` and `.bashrc` source all root-level plugins on startup; macOS-exclusive plugins live under `.profile.d/macos/` and are only loaded on Darwin. `.zshrc.local` (not tracked in git) can override or extend them. Each plugin is self-contained — to disable a feature, comment out its source line in `.zshrc.local`.

Notable plugins:
- `skeegfunctions.plugin.sh` — the largest file (~8300 lines), contains the bulk of custom shell functions
- `prompt.plugin.sh` — Starship cross-shell prompt (replaces oh-my-zsh + powerlevel10k)
- `fzf.plugin.sh` — fuzzy finder: Ctrl+R history, Ctrl+T file search, Alt+C directory jump
- `completions.plugin.sh` — zsh compinit + bash-completion setup
- `colors.plugin.sh` — colored man pages (bat), ls/grep/diff colors, zsh-autosuggestions, zsh-syntax-highlighting
- `aws.plugin.sh` — EC2 search, SSM sessions, Windows password retrieval from AWS
- `terraform.plugin.sh` — workspace aliases, validation helpers, interactive workspace selector
- `kubernetes.plugin.sh` — kube context switching aliases
- `docker.plugin.sh` — docker/compose convenience wrappers
- `macos/functions.plugin.sh` — macOS-specific utilities (network, system, Finder integration)

### Environment Layering

1. `.zshenv` — exports PATH and environment variables (loaded for all ZSH invocations)
2. `.zshrc` — sources all `.profile.d/` plugins (no framework dependency)
3. `.zshrc.local` — machine-local overrides (not tracked; generated by `bootstrap.sh` if missing)
4. `.bash_profile` — bash compatibility shim that sources profile.d plugins

### Git Aliases

Defined in `.profile.d/merge_content/.gitconfig_aliases` and merged into `~/.gitconfig` via `merge_gitconfig.sh`. Key aliases: `save` (stage-all + signed commit), `refresh` (rebase on upstream), `lg` (pretty log), `create` (pull + new branch).

### Secrets and Local Overrides

The `.gitignore` excludes `.npmrc`, `.yarnrc*`, and similar credential files. A `secret.sh` pattern (referenced in commented code) is the intended place for machine-specific secrets — keep that file outside this repo.
